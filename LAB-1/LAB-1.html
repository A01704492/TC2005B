El internet es la infraestructura que permite la comunicación global entre dispositivos, mientras que la World Wide Web (WWW) es una colección de páginas web que se acceden desde un navegador web los cuales están interconectados a través de enlaces hipertexto que permiten que los usuarios navegen y accedan a la información.

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Son comandos que permiten realizar diferentes acciones en un servidor web.

1. GET permite solicitar un recurso en específico en el servidor. Se envián datos a través de un URL y el servidor responde con los datos solicitados.

2. HEAD solicita los mismos datos que GET pero sin el cuerpo de respuesta.

3. POST envía datos al servidor en el cuerpo de la solicitud y pueden utilizarse para actualizar bases de datos o crear nuevos recursos.

4. PUT actualiza un recurso existente en el servidor.

5. PATCH permite actualizar parcialmente un recurso en el servidor.

6. DELETE elimina un recurso específico del servidor.

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Dado que el error 404 indica que la solicitud no pudo ser procesada debido a que el recurso solicitado no existe o no está disponible en la ubicación específicada. Por lo tanto es la responsabilidad del desarrollador restaurar la disponibilidad de la página.

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si, el error 500 indica un error interno en el cual el servidor no ha podido procesar la solicitud y se le solicita al usuario intentar más tarde. Es el desarrollador quien debe investigar el problema y tomar las medidas necesarias para resolver el problema.

11. ¿Qué componentes de estructura y estilo tiene una tabla?

1. 'table' - el cual crea la tabla.

2. 'tr' - define cada fila de la tabla.

3. 'td' - define cada celda de la tabla.

4. 'th' - sirve para definir el encabezado en una tabla.

5. 'caption' - define un título o leyenda para la tabla.

6. Atributos de Tabla - brindan el diseño de la tabla.

12. ¿Cuáles son los principales controles de una forma HTML5?

Los controles son utilizados en formularios que permiten a usuarios ingersar y enviar información

1. input type = 'text' - permite a los usuarios ingresar texto

2. input type = 'password' - permite a los usuarios ingresar una contraseña pero oculta el texto a medida que se escribe

3. input type = 'number' - permite ingresar numeros en campos de texto

4. input type = 'email' - permite a usuarios ingresar una dirección de correo electrónico

5. input type = 'tel' - permite a usuarios ingresar un número de teléfono

6. input type = 'checkbox' - permite que los usuarios seleccionen entre una o varias opciones de una lista de opciones

7. input type = 'radio' - permite a usuarios seleccionar UNA opcion de listas de opciones

8. 'select' - permite a usuarios seleccionar de una lista desplegable

9. 'textarea' - permite a usuarios ingresar texto en varías líneas de campo de texto

14. Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de vida de los sistemas de información es un proceso que sigue el desarrollo y mantenimiento de un sistema de información a los largo de su vida útil, consta de 5 etapas las cuales se desglosan de la siguiente forma:
1. Planificacón: Se definen objetivos y requisitos del sistema de información. A su vez se establecen los recursos necesarios y se elabora un plan de proyecto

2. Análisis: Se analiza el sistema actual y se identifican los procesos de negocio que deben de ser automatizados

3. Diseño: Se definen las específicaciones técnicas del sistema, se elabora el diseño de una base de datos y se desarrolla un modelo lógico y físico del sistema.

4. Implementación: Se lleva acabo la codificación del software y la instalación del hardware y software necesario. De igual forma se realiza la integración del sistema con otros sistemas.

5. Mantenimiento: Toma lugar el mantenimiento del sistema que puede incluir correción de errores, actualizaciones de docs., nuevas funcionalidades y mejoramiento del rendimiento.

El ciclo de vida de desarrollo de los sistemas de información brindan el enfoque hacia el ciclo de vida completo del sistema:
1. Identificación de requisitos: Se definen y se recompilan los requisitos del sistema de información. A su vez, se determinan los objtivos del sistema y los procesos que se busca automatizar.

Los pasos siguientes son exactamente los mismos que los del ciclo de vida de los sistemas de información.
